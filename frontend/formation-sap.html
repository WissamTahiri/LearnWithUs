SAP : le leader mondial des ERP

Un ERP (Enterprise Resource Planning, "progiciel de gestion intégré" en français) est un logiciel qui centralise la gestion de toutes les fonctions d'une entreprise : finance, achats, ventes, stocks, ressources humaines, production. Au lieu d'avoir un logiciel par métier, l'ERP offre une vision unifiée de l'activité en temps réel.

SAP est l'ERP le plus utilisé au monde. Fondée en 1972 en Allemagne par cinq anciens ingénieurs d'IBM, l'entreprise équipe aujourd'hui plus de 400 000 clients dans 180 pays, dont 92 % des entreprises du classement Forbes Global 2000.

Pourquoi apprendre SAP ?

  • Un marché de l'emploi porteur : les consultants SAP sont parmi les profils IT les plus recherchés, avec des salaires attractifs.
  • Un standard mondial : comprendre SAP, c'est comprendre comment fonctionnent les grandes organisations.
  • Une montée en compétences progressive : on peut commencer par les bases (navigation, un module métier) avant de se spécialiser.

Les grandes familles de modules

SAP est découpé en modules correspondant chacun à un métier. Les plus connus :

  • FI (Financial Accounting) : comptabilité générale.
  • CO (Controlling) : contrôle de gestion.
  • MM (Materials Management) : achats et gestion des stocks.
  • SD (Sales and Distribution) : ventes et expédition.
  • HR (Human Resources) : ressources humaines.
  • PP (Production Planning) : planification de la production.

Dans cette formation, nous couvrons les trois modules les plus fréquemment rencontrés : FI, MM et SD.

Naviguer dans SAP et comprendre ses modules clés

1. Se connecter et naviguer dans SAP GUI

SAP GUI est l'interface graphique historique pour accéder à SAP. La connexion se fait via un client local qui pointe vers un serveur d'entreprise. Chaque écran ou fonction est identifié par un code transaction appelé T-Code (ex : FB01 pour saisir une écriture comptable, ME21N pour créer une commande d'achat). Connaître les T-Codes est essentiel : ils permettent d'accéder directement à une fonction sans passer par le menu.

2. Module FI : la comptabilité

Le module FI gère la comptabilité générale, les comptes clients, les comptes fournisseurs, et les immobilisations. Il produit les états financiers officiels : bilan, compte de résultat, grand livre. Chaque écriture comptable est enregistrée dans un document qui suit le principe de partie double (débit = crédit). Les T-Codes phares : FB01 (saisie), FBL1N (extraction comptes fournisseurs), F-53 (paiement).

3. Module MM : les achats et les stocks

Le module MM couvre tout le cycle d'achat (Procure to Pay / P2P) :

  • Demande d'achat (besoin exprimé par un service).
  • Commande d'achat (envoyée au fournisseur).
  • Réception de marchandises (entrée en stock).
  • Réception facture (contrôle à 3 voies : commande / réception / facture).
  • Paiement (déclenché côté FI).

4. Module SD : les ventes

Le module SD couvre tout le cycle de vente (Order to Cash / O2C) :

  • Devis envoyé au client.
  • Commande de vente validée.
  • Livraison (sortie de stock).
  • Facturation client.
  • Encaissement (déclenché côté FI).

5. Intégration entre modules

La grande force de SAP est l'intégration. Une commande de vente dans SD génère automatiquement un mouvement de stock dans MM et une facture dans FI. Cette cohérence évite les ressaisies et garantit la fiabilité des données.

6. S/4HANA, la nouvelle génération

Depuis 2015, SAP déploie S/4HANA, sa nouvelle plateforme basée sur la base de données in-memory HANA (bien plus rapide) et une nouvelle interface web moderne appelée Fiori. Toutes les entreprises doivent migrer de l'ancienne version (ECC) vers S/4HANA d'ici 2030, ce qui génère un marché de migration considérable.
