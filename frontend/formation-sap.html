Pourquoi se former à SAP aujourd'hui ?

Derrière les rayons de supermarché, les paies des salariés, la comptabilité des multinationales : il y a souvent SAP. Fondé en 1972, ce géant allemand équipe aujourd'hui plus de 400 000 entreprises dans le monde, dont la quasi-totalité du Fortune 500. Résultat : les profils qui savent naviguer dans SAP sont parmi les plus recherchés — et les mieux payés — sur le marché de l'IT et de la gestion.

Dans cette formation, vous allez apprendre à :

  • Naviguer dans SAP GUI et utiliser les T-Codes les plus courants
  • Comprendre les modules FI (comptabilité), MM (achats) et SD (ventes)
  • Réaliser des transactions concrètes : saisie d'écriture, commande d'achat, consultation de stock

Transactions SAP courantes : 4 tutoriels pas à pas

La vidéo et les slides qui accompagnent cette formation présentent la vue d'ensemble de SAP : qu'est-ce qu'un ERP, les modules FI/MM/SD, le concept de T-Code et l'évolution vers S/4HANA. Cette partie écrite est un manuel pratique : 4 tutoriels détaillés pour réaliser les transactions les plus courantes que vous rencontrerez en stage ou en mission.

Tuto 1 — Saisir une écriture comptable (FB01)

  1. Dans la barre de commande SAP, tapez FB01 puis Entrée.
  2. Renseignez la date du document, la date comptable, le code société (souvent 4 caractères, ex : FR01) et la devise.
  3. Choisissez le type de document (SA pour une écriture diverse, KR pour une facture fournisseur).
  4. Saisissez la première ligne : code de mouvement (40 pour débit, 50 pour crédit), compte, montant, libellé.
  5. Ajoutez les lignes suivantes jusqu'à équilibrer débit = crédit, puis cliquez sur Sauvegarder (ou F11).

⚠️ Si vous voyez "Ecriture déséquilibrée" : vérifiez les codes 40/50 et les montants. La partie double est obligatoire.

Tuto 2 — Créer une commande d'achat (ME21N)

  1. Tapez ME21N dans la barre de commande.
  2. Sélectionnez le type de commande (NB pour commande standard) et le fournisseur.
  3. Dans l'en-tête, renseignez l'organisation d'achat, le groupe d'acheteurs et la société.
  4. Dans le tableau des postes, saisissez le code article, la quantité, le prix et le centre/magasin de livraison.
  5. Vérifiez la date de livraison prévue, puis cliquez sur Sauvegarder. SAP affiche un numéro de commande à 10 chiffres — à communiquer au fournisseur.

Tuto 3 — Vérifier le stock d'un article (MMBE)

  1. Tapez MMBE.
  2. Renseignez le code article (obligatoire). Vous pouvez filtrer par centre et magasin si vous cherchez un emplacement précis.
  3. Cliquez sur Exécuter (F8). SAP affiche les quantités disponibles par centre et par statut (disponible, en contrôle qualité, bloqué).
  4. Double-cliquez sur une ligne pour voir le détail par lot / emplacement / date d'entrée.

💡 Astuce : si vous ne connaissez pas le code article, utilisez MM03 pour le chercher par libellé.

Tuto 4 — Consulter une fiche client (VD03)

  1. Tapez VD03.
  2. Saisissez le numéro du client et la société. Cliquez Entrée.
  3. Naviguez entre les onglets : Adresse (coordonnées), Contrôle (catégorie de compte, classification), Paiement (conditions de paiement, moyens acceptés), Interlocuteurs (personnes à contacter).
  4. Pour voir les commandes passées par ce client, utilisez ensuite VA05 (liste des commandes de vente).

Astuces de productivité SAP

  • Favoris : faites un clic droit sur un T-Code dans le menu puis "Ajouter aux favoris". Vous y accéderez en 1 clic.
  • Variantes d'écran : après avoir configuré un écran (filtres, colonnes…), sauvegardez la configuration en variante pour la retrouver à chaque session.
  • Nouvelle fenêtre : tapez /O suivi d'un T-Code pour ouvrir une seconde session en parallèle sans perdre la première.
  • Retour rapide : /N revient au menu principal, /NEX ferme la session.

Pièges fréquents des nouveaux utilisateurs

  • Confondre date du document et date comptable : la première est celle du justificatif, la seconde est celle d'imputation dans la comptabilité.
  • Oublier l'organisation d'achat / société : SAP rejette la transaction. Toujours vérifier ces champs en premier.
  • Quitter sans sauvegarder : SAP ne prévient pas toujours. Prenez l'habitude de F11 avant de changer d'écran.
  • Travailler en production au lieu du mandant test : une écriture mal saisie en production est difficile à annuler. Validez toujours le mandant en haut à droite (ex : 400 = test, 100 = prod).
